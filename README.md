# 🤖 Email Agent - Автоматический сборщик контактов из писем

[![Python](https://img.shields.io/badge/Python-3.7%2B-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

Email Agent — это интеллектуальный помощник, который автоматически отслеживает входящие письма, извлекает из них контактные данные (телефон, email, адрес) и сохраняет их в удобную базу данных или Google Таблицы.

## ✨ Возможности

- 🔍 **Автоматический мониторинг** — проверяет непрочитанные письма каждые 30 секунд
- 🧠 **Умное извлечение данных** — находит в тексте письма:
  - 📞 Номера телефонов (российские форматы: +7, 8)
  - 📧 Email-адреса клиентов
  - 📍 Физические адреса (улица, дом, город)
- 💾 **Два режима хранения данных**:
  - **Локальный**: SQLite база данных + CSV файл (для Excel)
  - **Облачный**: Google Sheets (доступ из любого места)
- 🔐 **Безопасное хранение паролей** — учетные данные сохраняются локально в зашифрованном виде
- 📊 **Статистика работы** — показывает количество собранных контактов
- 🚀 **Простой запуск** — работает "из коробки" после минимальной настройки

## 🔄 Как это работает

┌─────────────┐ ┌──────────────┐ ┌─────────────┐ ┌──────────────┐
│ Почтовый │────▶│ IMAP │────▶│ Извлечение │────▶│ Сохранение │
│ ящик │ │ Подключение │ │ данных │ │ в БД/Таблицу│
└─────────────┘ └──────────────┘ └─────────────┘ └──────────────┘
text


1. Агент подключается к вашему почтовому ящику по протоколу IMAP
2. Находит все непрочитанные письма
3. Анализирует содержимое с помощью регулярных выражений
4. Извлекает структурированные данные
5. Сохраняет их в базу данных или Google Таблицу
6. Повторяет цикл каждые 30 секунд

## 📦 Требования

### Для локальной версии (SQLite + CSV):
- Python 3.7 или выше
- Доступ к почтовому ящику по IMAP
- **НЕ требует** установки дополнительных библиотек!

### Для облачной версии (Google Sheets):
- Python 3.7+
- Библиотеки: `gspread`, `oauth2client`
- Аккаунт Google Cloud Platform
- Включенные Google Sheets и Google Drive API

🔧 Конфигурация почтовых сервисов
Gmail

    Включите IMAP:

        Зайдите в Gmail → Настройки → "Пересылка и POP/IMAP"

        Включите "Доступ по протоколу IMAP"

    Создайте пароль приложения (ОБЯЗАТЕЛЬНО!):

        Перейдите в Аккаунт Google → Безопасность

        Включите "Двухфакторную аутентификацию"

        Выберите "Пароли приложений"

        Создайте пароль для "Почта" → "Другое"

        Скопируйте 16-значный пароль — он понадобится при настройке агента

    IMAP сервер: imap.gmail.com

Yandex

    Включите IMAP:

        Зайдите в Настройки → "Почтовые программы"

        Поставьте галочку "С сервера imap.yandex.ru по протоколу IMAP"

    Создайте пароль приложения:

        Перейдите в Яндекс ID → Безопасность

        Выберите "Пароли приложений" → "Создать пароль"

        Выберите "Почта" и создайте пароль

    IMAP сервер: imap.yandex.ru

Mail.ru

    Включите IMAP:

        Настройки → "Почтовые программы"

        Включите IMAP

    Создайте пароль для внешнего приложения:

        Настройки → "Пароли для внешних приложений"

        Создайте новый пароль

    IMAP сервер: imap.mail.ru

🔍 Устранение неполадок
❌ "Не удается подключиться к серверу"

Решение:

    Проверьте правильность IMAP сервера

    Убедитесь, что IMAP включен в настройках почты

    Для Gmail — проверьте, что используете пароль приложения, а не обычный

❌ "Нет новых писем" (хотя они есть)

Решение:

    Агент проверяет только НЕПРОЧИТАННЫЕ письма

    Отправьте себе тестовое письмо и не открывайте его в почтовом клиенте

❌ "Не извлекаются данные из писем"

Решение:

    Проверьте формат данных в письме

    Текущая версия ищет:

        Телефоны: +7 XXX XXX-XX-XX, 8 (XXX) XXX-XX-XX

        Email: стандартный формат name@domain.com

        Адреса: строки с "Адрес:", "ул.", "г."

❌ Проблемы с установкой библиотек (gspread)

Решение 1 — Использовать зеркало:
bash

pip install -i https://mirror.yandex.ru/pypi/simple gspread oauth2client

Решение 2 — Увеличить таймаут:
bash

pip install --default-timeout=100 gspread oauth2client

Решение 3 — Использовать локальную версию без дополнительных библиотек

📝 Часто задаваемые вопросы

В: Работает ли агент с корпоративной почтой?
О: Да, если сервер поддерживает IMAP и у вас есть данные для подключения.

В: Можно ли запускать агента на сервере 24/7?
О: Да, рекомендуется использовать screen/tmux или systemd сервис.

В: Сколько писем может обработать агент?
О: Ограничений нет, но Gmail имеет лимит в 2500 писем в день через IMAP.

В: Безопасно ли хранить пароль в файле?
О: Файл хранится локально. Для повышенной безопасности можно использовать системный keyring.

В: Можно ли настроить фильтрацию писем?
О: Да, раскомментируйте строку с mail.store(e_id, '+FLAGS', '\\Seen') чтобы помечать обработанные письма как прочитанные.
